Mistral 7B：滑动窗口与 GQA 的高效 decoder

主题：decoder-only、分组查询注意力、滑动窗口注意力、KV cache

对应技术报告：Albert Q. Jiang 等，Mistral 7B（2023）

解决什么问题

小型 decoder 模型常受两类推理成本限制：每层全注意力读取全部历史，KV cache 随序列长度线性膨胀；而每个 query 头都保留独立 K/V 头，也会增加带宽与显存。Mistral 7B 的目标是在约 7B 参数预算内，把注意力计算、KV cache 与上下文传递做得更有效率，而不是用更大稠密模型换性能。

原作者的方法与论证

报告结合三项设计。第一，Grouped-Query Attention（GQA）：多个 query heads 共享较少的 key/value heads，减少生成时读取的 KV cache。第二，Sliding Window Attention（SWA）：每层 token 只注意固定窗口内历史；论文采用滚动 KV cache，窗口之外的 K/V 不再保留，从而限制内存。第三，层层堆叠的局部窗口会扩大有效感受野：信息可在相邻层逐步向后传递，尽管单层并不直接看完整历史。模型使用 RoPE 编码位置。

报告以 32k token 的例子说明，SWA 比全注意力显著减少 cache；它同时强调这不是“旧 token 完全没有影响”，而是旧信息要经由中间 token 的层间传播才能到达当前位置。GQA 降低 KV 头数，主要针对自回归推理的内存带宽瓶颈。

核心机制

多个 Q heads 共享较少 K/V heads 降低缓存体积；每层仅读滑动窗口 限制注意力与 KV cache；多层堆叠 再把局部信息逐层向远处传播。三者共同改变的是实际 decode 成本，不是模型的因果生成方式。

实验与适用场景

作者在常识、阅读理解、数学、代码、世界知识与多任务评测上比较 Llama 2 等基线，并报告 Mistral 7B 在所列基准上优于 Llama 2 13B、在若干指标接近或超过更大模型。报告也测试长上下文检索任务，说明训练时引入 8k 上下文与滑动窗口可处理较长输入。

适合 GPU 显存和生成吞吐敏感的通用对话、代码或文本任务。若任务需要对远处原始 token 做单层、精确的随机访问，SWA 的局部连接可能不如全注意力；应测试真实上下文分布而非只看标称窗口。

整理者归纳

Mistral 7B 展示了一个重要工程原则：上下文长度由三件事共同决定，位置编码能表示多远、每层 attention 能读多远、KV cache 能承受多远。它们不是同一个数。SWA 与 RoPE 解决不同层面，GQA 则主要压低 KV 成本。模型选型时要测首 token 延迟、decode 吞吐、有效信息距离和长文任务质量。

限制与关系

  • GQA 是压缩 K/V 共享，不会消除自回归生成按 token 串行的本质。
  • 窗口传播有路径长度；层数不足或信息被压缩时，远处细节可能衰减。
  • 与 RoPE：把相对位置信息旋进注意力计算：Mistral 使用 RoPE；但 RoPE 解决位置表示，SWA 解决每层读取范围。
  • 与 Mamba：用选择性状态空间模型线性处理长序列：两者都缓解长序列成本；前者保留局部 attention，后者改用选择性状态递推。
