DeepSeek LLM：把 scaling law 变成 7B 与 67B 的训练决策

主题：预训练、Scaling Laws、数据质量、SFT、DPO

对应材料：DeepSeek-AI，DeepSeek LLM: Scaling Open-Source Language Models with Longtermism（arXiv:2401.02954，2024）

这篇报告解决了什么问题

开源模型常沿用固定的 7B、13B、70B 配置，却未必知道在有限算力下模型大小、训练 token、batch size 和学习率怎样配才合算。报告的目标是把缩放规律落到两个实际配置：7B 与 67B，并用此指导数据建设、预训练和后训练，而非只报告一个最终模型分数。

原作者的论证、方法与核心机制

作者先研究超参数随模型规模的变化，再估计在给定计算预算下模型和数据的最优分配；一个重要观察是，使用不同数据集得到的 scaling curve 会显著不同。因此论文明确反对把单一数据集上的幂律无条件推广为通用定律。

在实现上，模型基本沿用 LLaMA 式 decoder-only Transformer：RMSNorm、SwiGLU、RoPE；67B 用 GQA 降低生成缓存，并使用多阶段学习率调度以利继续训练。数据管线分去重、过滤、重混合三个阶段：跨 91 个 Common Crawl dump 的全局去重比单个 dump 去重移除了更多重复文档，随后用质量过滤提高信息密度，再调节语种和领域比例。报告称其预训练集为 2T 中英 token。

后训练部分以超过 100 万条样本做 SFT，再以 DPO 改善对话偏好。这里的结构很重要：缩放规律决定“训练资源投向何处”，而 SFT/DPO 决定“用户如何感知模型”；两层指标不能混成一个结论。

实验或适用场景

原作者报告 67B base 在多项代码、数学、推理基准超过 LLaMA 2 70B，67B Chat 在其开放式中英文评测中优于 GPT-3.5。更可迁移的实验信息是数据消融：论文将数据构成视为会改变 scaling behavior 的变量。适合该方法的场景是团队要在固定预算下训练一条长期模型线，而不是只为一次发布选择最大参数量。

整理者归纳

这份报告把 scaling law 从“预测 loss 的图”转为工程控制面板：先在自己的数据、目标上下文和训练栈上做小规模曲线，再决定模型和数据比例。它也提示知识库或领域模型项目：数据去重和重混合不是预处理杂务，数据分布改变时，历史上的最优训练配方可能失效。

限制

该报告的规律来自其数据、tokenizer、模型结构与预算，不能替代当前训练硬件和数据上的测量。基准与开放式比较主要是作者的评估设置；没有完整公开原始数据和标注细节。2T token、7B/67B 的结论也不直接回答 MoE、长上下文或推理 RL 的资源分配问题。

与现有概念的关系

  • Scaling Laws：本报告提供了“不同数据集会改变规律”的实际反例与应用路径。
  • Transformer：用自注意力重写序列建模：该系列建立在 decoder-only Transformer 与 RoPE/GQA 等变体上。
  • InstructGPT：SFT 如何让预训练模型开始遵循指令：SFT/DPO 是预训练后把基础能力转换为可对话行为的层。
