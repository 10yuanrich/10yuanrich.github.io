Mamba：用选择性状态空间模型线性处理长序列

主题：状态空间模型、选择性机制、线性序列建模、长上下文

对应论文：Albert Gu、Tri Dao，Mamba（2023）

解决什么问题

Transformer 的全注意力让任意 token 直接交互，却在长序列上付出二次计算和 KV cache 成本。此前状态空间模型（SSM）可线性扫描序列，但在语言等离散内容任务上常输给 attention：固定的状态更新无法根据当前 token 决定哪些信息应记住、忽略或取回。Mamba 的问题是：能否给 SSM 内容选择能力，又不丢掉线性时间和高吞吐。

原作者的方法与论证

连续 SSM 可写成隐藏状态随输入递推、再读出输出的系统。传统高效实现让离散化步长和输入/输出投影不随 token 改变，因此很像固定滤波器。Mamba 让这些参数成为输入 x 的函数：当前 token 可以改变更新步长、写入状态的内容和读出的方式。作者称之为 selective SSM；它使模型能选择性传播或遗忘信息。

这种输入依赖破坏了传统 SSM 的卷积并行化。论文于是设计硬件感知的 parallel scan，在递推视角下保留线性计算与并行执行；模型块还将选择性 SSM、门控、归一化和投影组合成端到端 Mamba，去除了 attention，论文版本也不使用传统 Transformer MLP 块。

核心机制

当前 token -> 生成本步 SSM 参数 -> 更新压缩状态 -> 从状态读出输出。选择性在于每步参数随输入变动，scan 在于把这条递推以硬件友好的并行前缀计算执行；前者解决内容依赖，后者保留效率。

实验与适用场景

作者在复制、选择性复制、induction 等合成任务上检验内容选择能力，并在语言、音频、基因组等序列任务上评估。摘要报告 Mamba-3B 超过同规模 Transformer、匹配约两倍规模 Transformer 的语言模型表现；同时在其实现与基准上，推理吞吐可高于 Transformer，长度扩展保持线性。这里的实验支持其作为通用序列骨干的潜力，不等于所有 LLM 任务的替代结论。

当输入极长、流式处理、缓存空间或每 token 延迟是约束时，SSM 路线尤其值得比较。若任务强依赖对任意历史片段做精确、动态的多对多关联，应单独验证其是否达到 attention 的效果。

整理者归纳

Mamba 的核心不是“RNN 回归”，而是把可学习记忆变为按内容调节的递推状态。它将检索式的“看哪段上下文”换成“保留什么状态”；后者更省，但状态是压缩摘要，不能像 attention 一样随时显式回看每个 token。架构选择应先问信息访问模式，而不是只比较渐近复杂度。

限制与关系

  • 线性复杂度不自动带来端到端更快；scan 内核、批量、硬件和训练实现决定实际优势。
  • 选择性状态容量有限，长程精确引用或证据追溯可能不如显式检索/attention 直接。
  • 与 Transformer：用自注意力重写序列建模：Mamba 以状态扫描替代全局 attention，目标是改变长序列成本结构。
  • 与 检索增强生成：二者都针对有限上下文与知识访问；Mamba 压缩内部历史，RAG 从外部文档显式取证，不能互相等同。
